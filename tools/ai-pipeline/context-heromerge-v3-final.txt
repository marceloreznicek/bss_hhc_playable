## RESUMO DAS 4 MUDANÇAS
1. **3-of-a-kind merge**: `tryMergeCells` chama `computeMergeCluster(src, drop)` → cluster de 3, todos consumidos, próximo tier no drop.
2. **Adjacency required**: `computeMergeCluster` exige que `dropIdx` tenha pelo menos 1 vizinho geometricamente adjacente (via `GRID_ADJ` precomputado pro grid 4×2) que seja mesmo tier e ≠ srcIdx. Spec do user: "2 das 3 lado a lado" → drop+partner adjacentes satisfaz isso.
3. **Pulsing magnetic-pull preview**: `updatePullPreview` (rAF-coalesced em pointermove) marca `.pulling` em exatamente as 3 células do cluster. CSS adiciona scale(1.09) + golden ring expansivo + glow. Cleanup em endDrag.
4. **No overconsume**: cluster sempre = 3 elementos. 4ª célula de mesmo tier fica intacta no grid (validado: snap_grid mostra cell 2 = T1 após merge de [3,1,0]).
5. **Ghost offset fix**: `#drag-ghost` movido pra `document.body` (fora do `#game` que tem `transform:scale`), `position:fixed`, `transform:translate(-50%,-50%) scale(1.15)`. `moveDragGhost(clientX, clientY)` seta left/top diretamente em viewport pixels. Resultado: ghost embaixo do dedo, sem offset.

Defesas adicionais (precode synthesis):
- `CELL_ELS` cache (8 nós, 1× no init)
- Single-pointer lock via `DRAG.active`
- TOCTOU revalidation em `tryMergeCells` antes de mutar
- Cleanup `.pulling`/`.match`/`.dragging` em pointerup, pointercancel, lostpointercapture, visibilitychange, e início de novo drag
- `clusterEquals()` evita repintura DOM redundante quando finger se move dentro da mesma célula
- `setPointerCapture` (try/catch) pra dedo que sai do grid
