## GDD CONSOLIDADO (HeroMerge v0.1)

### Conceito (1 parágrafo)

Tap **SUMMON** dropa Recruits Tier-1 no campo. **Drag** uma unit em cima de
outra do mesmo tier → mescla em tier maior. Ao tocar uma unit, todas do
mesmo tier num raio de ~350px são puxadas magneticamente em direção ao dedo.
Ao soltar (ou após 0.6s parado), o cluster funde em UMA unit do próximo
tier no ponto da soltada. Combate auto. Sobreviver as 3 waves do HeroSwarm
e matar o Demon Lord = vitória.

|---|---|---|---|---|---|
| 1 | Recruit    | hero    (intro-hero.png)  | 5   | 1  | 110 |
| 2 | Lumberjack | ogre    (lenhador.png)    | 10  | 3  | 70  |
| 3 | Knight     | knight  (igris.png)       | 20  | 6  | 100 |

(Stats vêm direto das tabelas existentes — apenas reuso.)

### Mecânica magnética — comportamento exato

- Player toca uma unit → guarda `STATE.heldActor`
- Cada frame: outras units do MESMO tier dentro do raio R=350px lerpam em
  direção ao `heldActor` na velocidade ~600 px/s (ease-out)
- `heldActor` segue o dedo (pointer x/y → game coords)
- Ao soltar (pointerup) OU após 600ms de hold-still:
  - Identifica o cluster: heldActor + todas units do mesmo tier dentro de
    R_MERGE=80px da posição do dedo
  - Se cluster.length >= 2: despawn todas + spawn 1 unit do próximo tier no
    ponto do dedo + flash + SFX
  - Se cluster.length < 2: heldActor volta pro lugar (cancelar)

### SUMMON button (substitui as 3 cards)

- 1 botão grande na parte de baixo da tela
- Cooldown ~700ms (estado visual: cd-sweep + cd-text)
- Spawna 1 Recruit (Tier 1) em posição aleatória dentro do bottom half
  do battlefield (mesmo `bottomHalfY` usado pra enemies)
- Pode ser tapado a qualquer momento, mesmo durante drag-merge

### Tutorial (3 passos)

2. **Após 2 Recruits estarem vivos**: bubble "DRAG TO MERGE!" + guide hand

3. **Após 1ª merge feita**: tutorial off

### 5. Atração magnética (~50 LoC, no game loop)

```js
function tickMagnetic(dt) {
  var h = STATE.heldActor;
  if (!h) return;
  var R = 350, R2 = R * R;
  for (var i = 0; i < STATE.units.length; i++) {
    var u = STATE.units[i];
    if (u === h || u.tier !== h.tier || u.dead) continue;
    var dx = STATE.pointerX - u.x, dy = STATE.pointerY - u.y;
    var d2 = dx*dx + dy*dy;
    if (d2 > R2) continue;
    var d = Math.sqrt(d2);
    if (d < 1) continue;
    var pull = Math.min(600 * dt, d);  // px/s, never overshoot
    u.x += (dx / d) * pull;
    u.y += (dy / d) * pull;
    applyActorTransform(u);
  }
  // Auto-fire merge if held still for >600ms
  if (performance.now() - STATE.holdStartedAt > 600) {
    resolveMergeAtPointer();
    STATE.heldActor = null;
  }
}
```

### 6. Merge resolution (~40 LoC)

```js
function resolveMergeAtPointer() {
  var h = STATE.heldActor;
  if (!h) return;
  var R_MERGE = 80, R2 = R_MERGE * R_MERGE;
  var cluster = [h];
  for (var i = 0; i < STATE.units.length; i++) {
    var u = STATE.units[i];
    if (u === h || u.tier !== h.tier || u.dead) continue;
    var dx = STATE.pointerX - u.x, dy = STATE.pointerY - u.y;
    if (dx*dx + dy*dy < R2) cluster.push(u);
  }
  if (cluster.length < 2) {
    h.state = 'idle';
    return;  // cancelado, fica onde está
  }
  // Despawn cluster
  cluster.forEach(function(u) { u.dead = true; u.dom.remove(); });
  STATE.units = STATE.units.filter(function(u) { return !u.dead; });
  // Spawn next tier no ponto do dedo
  var nextKey = TIER_TYPE[h.tier];  // tier 1 → index 1 → ogre
  if (!nextKey) return;             // já tava no tier 5
  var newU = createActor(nextKey, UNIT_TYPES, STATE.pointerX, STATE.pointerY);
  newU.tier = h.tier + 1;
  STATE.units.push(newU);
  playSFX('unlock');  // power up sound
  // TODO: flash visual no spawn point (CSS .merge-flash)
}
```

# Revisão Claude Opus 4.7 — HeroMerge Pré-Code

## 🔴 BUGS CRÍTICOS (quebram o jogo)

### 1. Auto-fire vs pointerup — double-merge / crash
Em `tickMagnetic`, se `holdStartedAt` expira **no mesmo frame** do pointerup, `resolveMergeAtPointer()` roda 2× e `STATE.heldActor` já é null na segunda. Pior: cluster pode tentar despawnar actors já removidos.
**Fix**: guarda `if (!STATE.heldActor) return;` no início de `resolveMergeAtPointer`, e limpa `holdStartedAt = 0` após resolver.

### 2. `moveActors` continua movendo o heldActor
O game loop mantém `moveActors(dt)` rodando. O heldActor vai ser puxado pelo AI enquanto o pointer-move seta `x/y` — briga entre target-seek e dedo, gerando jitter.
**Fix**: em `moveActors`, `if (u === STATE.heldActor || u.state === 'held') continue;`

### 3. `tickTackles` + dano no heldActor
Unit segurada ainda recebe dano/lunge. Pior: pode **morrer durante o drag** → `h.dead=true`, mas `STATE.heldActor` ainda aponta pra ela. Pointermove continua atualizando `h.x/y` em actor morto/removido do DOM.
**Fix**: watcher em `tickTackles` e no damage resolver: se `actor === STATE.heldActor` e `hp<=0`, force clear `STATE.heldActor = null` + cancel state.

### 4. `createActor` pode não inicializar `tier`
Enemies passam por `createActor(typeKey, ENEMY_TYPES, ...)` — proposta diz "tier=-1 para enemies", mas o código mostrado em (2) só seta `recruit.tier=1` depois do createActor. Se o merge spawn esquecer de setar tier ANTES do próximo frame de `tickMagnetic`, a nova unit é puxada com `tier=undefined`.
**Fix**: centralizar `a.tier = computeTier(typeKey, registry)` DENTRO de createActor.

### 5. Cluster inclui units fora do cone visual / atrás do dedo
`resolveMergeAtPointer` usa R_MERGE=80 do pointer, mas units magnéticas podem estar a 100-300px ainda voando em direção ao dedo. Jogador vê 4 units "quase lá" e merge só pega 2 → sensação de bug.
**Fix sugerido**: ou (a) aumentar R_MERGE pra 150, ou (b) no auto-fire, esperar cluster stabilizar (todas units dentro de R_MERGE) antes de disparar.

## 🟡 EDGE CASES

6. **Level transition durante hold**: `STATE.gameActive=false` mas `heldActor` persiste. Se wave 2 resetar `STATE.units`, heldActor aponta pra objeto órfão. → Force `STATE.heldActor=null` em todo transition hook.

7. **Tier 5 no heldActor**: guard `hit.tier >= 5` impede pickup, mas não impede que uma Tier 5 seja **puxada magneticamente** por outra sendo held? Não — guard por `u.tier !== h.tier` cobre, ok. Mas confirme que não há ladder tier 5→6.

8. **`findActorAt` não mostrado**: precisa iterar units+enemies? Só units com `tier!=null`. Defina hit priority (topmost z-index / últimos em array).

9. **Drag fora do battlefield**: pointer pode ir pra cima do wave-info/summon. Clampe `pointerX/Y` aos bounds de FIELD antes de usar em magnetic/merge.

10. **Multi-touch**: segundo dedo dispara outro pointerdown → sobrescreve `heldActor`, primeiro dedo fica zumbi. Use `pointerId` lock.

# Auditoria de Segurança — HeroMerge Playable

**Papel: SEGURANÇA | Vetor de ataque em HTML5/JS client-side sem servidor**

---

## Contexto crítico

Este é um **playable de anúncio HTML5** — single-file, sem backend, sem IAP real, sem Firebase. O modelo de ameaça é completamente diferente do jogo Unity mobile. Não há receipts, rubies persistidos ou torneios. Redireciono o foco para os vetores reais deste ambiente.

---

## Vetores de Ataque Relevantes

### 1. Spam de SUMMON via console / automação

**Vetor:** `STATE.cooldownUntil.summon` é uma variável JS exposta globalmente. No browser, qualquer usuário abre DevTools e executa:
```js
STATE.cooldownUntil.summon = 0;
// Loop:
setInterval(() => { STATE.cooldownUntil.summon = 0; onSummonTap(); }, 10);
```
**Resultado:** 500+ units no DOM em segundos → travamento/crash do browser. Para um *playable de anúncio*, isso afeta a **métrica de engajamento** (sessão longa/anômala) e pode crashar o iframe do ad network, gerando chargeback de impressão.

**Mitigação:** Cap hard no servidor não existe aqui, então use cap local intocável:
```js
var MAX_PLAYER_UNITS = 20; // constante fora do STATE
if (STATE.units.length >= MAX_PLAYER_UNITS) return;
```

---

### 2. Multitouch / ponteiros múltiplos quebram `STATE.heldActor`

**Vetor:** Dois dedos simultâneos no mobile. `pointerdown` dispara duas vezes — o segundo sobrescreve `STATE.heldActor` sem liberar o primeiro. O ator original fica "fantasma": removido do controle do loop mas sem `pointerup` para resolver o merge.

**Resultado:** Unit presa em estado `'held'` indefinidamente, immune a `moveActors` e `resolveStacking`, mas ainda colidindo. Com `dead = false`, nunca é removida do `STATE.units`.

**Mitigação:**
```js
// No pointerdown:
if (STATE.heldActor) return; // rejeita segundo ponteiro
// Ou rastrear pointerId:
STATE.heldPointerId = e.pointerId;
// No pointermove/pointerup: ignorar se e.pointerId !== STATE.heldPointerId
```

---

### 3. Unit morre durante drag — estado zumbi

**Vetor:** `heldActor` recebe dano fatal de um enemy enquanto está sendo arrastado. `tickTackles` marca `u.dead = true` e chama `u.dom.remove()`. O `STATE.heldActor` ainda aponta para o objeto JS com `dead = true` e DOM removido.

**Resultado:** `resolveMergeAtPointer` executa no `pointerup`, tenta acessar `h.dom` (null) e `cluster` inclui o morto → `TypeError` + possível spawn de unit do próximo tier **de graça** (merge de 1 unit morta).

**Mitigação:**
```js
function resolveMergeAtPointer() {
  var h = STATE.heldActor;
  if (!h || h.dead) { STATE.heldActor = null; return; } // guard obrigatório
  // ...
  cluster.forEach(function(u) {
    if (!u.dead) { u.dead = true; u.dom.remove(); } // idempotente
  });
}
```

---

### 4. Auto-fire + pointerup — merge duplo

**Vetor (race condition de segurança):** O timer de 600ms de hold-still dispara `resolveMergeAtPointer()` no mesmo frame que `pointerup` também a chama. Dois merges executam sobre o mesmo cluster.

**Resultado:** O cluster é despawnado duas vezes (segunda iteração filtra units já `dead`, OK), mas **duas units do próximo tier são criadas** no mesmo ponto — duplicação gratuita de unidades.

**Mitigação:** Flag de guarda:
```js
function resolveMergeAtPointer() {
  if (STATE.mergeInProgress) return;
  STATE.mergeInProgress = true;
  // ... lógica ...
  STATE.mergeInProgress = false;
  STATE.heldActor = null;
}
// No auto-fire E no pointerup: chamar resolveMergeAtPointer() → internamente idempotente
```

---

### 5. Drag fora do viewport — coord injection

**Vetor:** `toGameCoords(e)` com `e.clientX` negativo ou > viewport. `STATE.pointerX` pode ser `-9999`. A unit spawna fora do campo visível mas permanece em `STATE.units`, participando de combate invisível e sendo contada em merges.

**Mitigação:**
```js
function clampToField(x, y) {
  return {
    x: Math.max(0, Math.min(LANE_W, x)),
    y: Math.max(FIELD_PAD_TOP, Math.min(FIELD_PAD_BOTTOM, y))
  };
}
// Aplicar no pointerup antes de resolveMergeAtPointer
```

function tickMagnetic(dt) {
  var h = STATE.heldActor;
  if (!h || h.dead || !STATE.gameActive) return;

  var R = 350;
  var R2 = R * R;
  var stepMax = 600 * dt;

  for (var i = 0; i < STATE.units.length; i++) {
    var u = STATE.units[i];
    if (u === h || u.dead || u.dying || u.tier !== h.tier) continue;

    var dx = STATE.pointerX - u.x;
    var dy = STATE.pointerY - u.y;
    var d2 = dx * dx + dy * dy;
    if (d2 > R2 || d2 < 1) continue;

    var d = Math.sqrt(d2);
    var pull = Math.min(stepMax, d);
    u.x += dx / d * pull;
    u.y += dy / d * pull;
    u.y = Math.max(FIELD_PAD_TOP, Math.min(FIELD_PAD_BOTTOM, u.y));
    applyActorTransform(u);
  }

  if (performance.now() - STATE.holdStillSince > 600) {
    resolveMergeAtPointer();
  }
}

### 5. Merge resolution robusto

Evitar tier 5, dead durante hold, double-call, game inactive, pointerup depois do auto-fire.

```js
function clearHeld() {
  if (STATE.heldActor) STATE.heldActor.state = 'idle';
  STATE.heldActor = null;
  STATE.heldPointerId = null;
  STATE.mergeResolving = false;
}

function resolveMergeAtPointer() {
  var h = STATE.heldActor;
  if (!h || STATE.mergeResolving) return;
  STATE.mergeResolving = true;

  if (!STATE.gameActive || h.dead || h.tier == null || h.tier >= 5) {
    clearHeld();
    return;
  }

  var R = 80;
  var R2 = R * R;
  var cluster = [h];

  for (var i = 0; i < STATE.units.length; i++) {
    var u = STATE.units[i];
    if (u === h || u.dead || u.dying || u.tier !== h.tier) continue;
    var dx = STATE.pointerX - u.x;
    var dy = STATE.pointerY - u.y;
    if (dx * dx + dy * dy <= R2) cluster.push(u);
  }

  if (cluster.length < 2) {
    h.x = h.dragHomeX;
    h.y = h.dragHomeY;
    applyActorTransform(h);
    clearHeld();
    return;
  }

  var nextKey = TIER_TYPE[h.tier];
  var nx = STATE.pointerX;
  var ny = Math.max(FIELD_PAD_TOP, Math.min(FIELD_PAD_BOTTOM, STATE.pointerY));

  for (i = 0; i < cluster.length; i++) {
    cluster[i].dead = true;
    if (cluster[i].dom && cluster[i].dom.parentNode) {
      cluster[i].dom.parentNode.removeChild(cluster[i].dom);
    }
  }

  STATE.units = STATE.units.filter(function(u) { return !u.dead; });

  var nu = createActor(nextKey, MERGE_UNIT_TYPES, nx, ny);
  nu.tier = h.tier + 1;
  STATE.units.push(nu);

  STATE.mergeCount++;
  playSFX('unlock');
  clearHeld();
}
```

### Edge Cases Críticos

**1. Pointer events em mobile**
- `pointerdown` no `#lane` pode não capturar se o sprite tiver `pointer-events: none` (comum em CSS de sprites). **Solução:** usar `e.target.closest('.actor')` ou hit-test manual via coordenadas.
- Multitoque: `pointerdown` com 2 dedos → segundo dedo pode selecionar outra unit. **Solução:** `if (STATE.heldActor) return;` no início do handler.

**2. Z-index do held actor**
- Durante drag, o actor precisa ficar acima de tudo. **Solução:** `heldActor.dom.style.zIndex = 9999;` no `pointerdown`, restaurar no `pointerup`.

**3. Hit area vs sprite visual**
- Sprites têm `visualScale: 1.89` mas hitbox é `w/h` original. Unit tier-5 (dragon) tem `w:342` → hit area enorme. Merge radius 80px pode ser menor que a própria unit. **Solução:** usar `R_MERGE = Math.max(80, hit.w * 0.5)`.

**4. Game loop vs drag**
- `moveActors(dt)` ainda move units `held`? **Solução:** pular units com `state === 'held'` no `moveActors`.
- `tickTackles` aplica dano em unit segurada? **Solução:** mesma verificação.

**5. Unit morre durante cluster**
- `resolveMergeAtPointer` itera `STATE.units` enquanto remove (`u.dom.remove()`). Se uma unit do cluster já está `dying` (animação 0.42s), o `dom.remove()` pode conflitar. **Solução:** marcar `u.dead = true` e deixar o `gameLoop` fazer cleanup via `STATE.units = STATE.units.filter(u => !u.dead)`.

A mecânica magnética é sensacional para o "pulo do gato" de satisfação tátil (ASMR), mas em um motor top-down com colisão, ela é um campo minado.

Aqui estão os insights de UX e estabilidade para o **HeroMerge**:

1.  **O Fantasma do Combate (Race Condition):** O maior risco é o `moveActors` e o `resolveStacking` continuarem brigando com o seu drag. Se uma unidade "magnética" entrar em combate ou for empurrada enquanto voa para o dedo, o movimento ficará tremido (jitter).
    *   **Solução:** Ao setar `u.state = 'held'` ou quando uma unidade entrar no raio de atração, você **precisa** flagar `u.isMagnetic = true` para ignorar o `moveActors` e o `resolveStacking` no loop principal.

2.  **O Problema da Hierarquia Visual (Z-Index):** No `HeroSwarm`, o `z-index` é `floor(y)`. Ao arrastar uma unidade, se ela passar "por trás" de um inimigo ou de outra unidade parada (Y maior), a ilusão de controle quebra.
    *   **Insight:** Unidades no `cluster` e o `heldActor` devem receber um `z-index: 9999` temporário e um `transform: scale(1.15)` para dar profundidade de "item levantado".

3.  **Abuso do Summoner (Performance Limit):** Sem custo de mana, o jogador vai espamar o botão. 100+ atores com `tickMagnetic` calculando distância quadrática (`dx*dx + dy*dy`) a cada frame em um mobile lento vai derrubar o FPS.
    *   **Pulo do gato:** Limite o `STATE.units.length`. Se atingir 20 unidades, desabilite o botão CSS (`grayscale + pointer-events: none`) com feedback visual "Full". Isso força o merge.

4.  **Feedback de "Fusão Iminente":** O `resolveMergeAtPointer` por distância pura (80px) é seco. O jogador precisa saber *quem* vai sumir antes de soltar o dedo.
    *   **UX:** Quando as unidades entrarem no `R_MERGE`, aplique um filtro CSS `brightness(1.5)` ou um leve `shake` nelas. Se soltar e nada acontecer, a sensação é de bug; se houver feedback visual, é erro do usuário.

5.  **Multi-touch e Ghosting:** No mobile, o jogador pode tentar arrastar dois grupos com dois dedos.
    *   **Fix rápido:** No `pointerdown`, adicione um check: `if (STATE.heldActor) return;`. Ignore o segundo toque para evitar que o `heldActor` pule entre dedos e quebre o estado do cluster.

  - Execução: Toque com 2 dedos em units diferentes do mesmo tier. O código atual sobrescreve STATE.holdStartedAt toda vez que qualquer pointermove acontece. Solte um dedo, segure o outro.
  - Impacto: O timer de 600ms nunca reseta corretamente, fazendo auto-merge disparar constantemente mesmo quando o jogador já soltou. Cria merge chain infinita até crash por stack depth no despawn.
