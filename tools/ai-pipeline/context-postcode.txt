## CONTEXTO DO BUG

Tela "TOP HERÓIS ÚLTIMO BOSS" do GlobalBossManager exibia:
- 1º Myth (SarcasticKhush): 161.55M dano
- 2º Ryota (sxRyota):       162.12M dano  ← MAIOR que o 1º!
- 3º Medallian (MedallianX):158.55M dano

O nó "Tops" no Firebase (https://hero-hero-clicker-world-boss-room.firebaseio.com/Tops.json) é populado por servidor externo (Unity headless em Oracle Cloud, código fora deste repo) na ordem dos índices [1], [2], [3] derivada do nó "Positions". Essas Positions estão dessincronizadas com o dano real (verificado lendo Players.json e Positions.json — pos=1 tem 163M, pos=2 tem 167M, pos=3 tem 179M, pos=4 tem 180M).

```csharp
private static List<string> ExtractTopValues(Dictionary<string, object> dict, int count)
{
    var result = new List<string>(count);
    if (dict == null || dict.Count == 0)
        return Enumerable.Repeat("null", count).ToList();
    var ordered = dict.Select(...).OrderBy(...).ThenBy(...);
    foreach (var e in ordered)
    {
        if (e.IsNum && e.Num == 0) continue; // pula índice 0 (placeholder)
        if (e.Value == null) continue;
        var s = e.Value.ToString();
        if (string.IsNullOrEmpty(s) || s == "0") continue;
        result.Add(s);
        if (result.Count >= count) break;
    }
    while (result.Count < count) result.Add("null");
    return result;
}
```
