## 2. SPEC RESUMIDO — Multiple Dungeons (v1)

**Objetivo:** 3 dungeons distintas; jogador escolhe qual rodar a cada sessão. Cada uma força composições/decisões diferentes via stat profile + regra de focus-targeting.

### Lineup v1 (final)

| ID | Nome | Unlock | Focus rule | Stat profile |
|---|---|---|---|---|
| `vault` | The Vault | sempre | **Standard** (toca enemy → party concentra fogo; sem focus → cada hero escolhe random) | balanceado, defaults |
| `hollow` | The Hollow | clear Vault Floor 10 | **No-Focus** (botão de focus DESABILITADO) | swarm: mais inimigos, HP menor cada |
| `drowned_crypt` | The Drowned Crypt | clear Vault Floor 20 | **Forced-Focus** (1 inimigo "Tide" por floor; se focus ≠ Tide enquanto Tide vivo → todos inimigos causam **+50% damage**) | elite-anchored: 1 anchor com HP×3 + 2–3 supports |

"Reached Floor N" = `deepestFloorByDungeon[id] >= N` (banked OR wiped, ambos contam — alcançou a floor é alcançou).

### Mudanças de fluxo
- App agora abre em **Dungeon Select** (canvas novo). 3 cards. Lock card = shake + tooltip.
- Card mostra **focus rule verbatim** + **pressure hint genérico**. SEM hints de comp/synergy (gated até existir Cartographer building).
- Jogador toca card unlocked → vai DIRETO pro Party Select (sem confirm panel — PM-default Q9). Banner do dungeon no Party Select.
- Bank/wipe → volta pra Dungeon Select (não pro Party Select).
- Per-dungeon **deepest floor** trackeado; gold é **shared wallet** (sem mudança).
- "New" badge em dungeon recém-unlocked, limpa após visualização ou após rodar a dungeon.

### Save migration (v1 → v2)
- `dungeonsUnlocked = { "vault" }` (default).
- `deepestFloorByDungeon = { "vault": existing global deepestFloor }` (preserva progresso).
- `seenJustUnlocked = {}`.
- `seenDungeonFocusExplanation = {}` (pra mostrar tooltip do No-Focus/Forced-Focus apenas na 1ª vez).
- Save corrupto → fallback graceful pra defaults (Vault-only). Não crashar.

### Tide enemy (Drowned Crypt) — determinismo
- Tide enemy index = função pura de `(dungeonId, floor, enemyCount)`. PM-default: `tideIdx = hash(dungeonId, floor) % enemyCount`.
- Anchor enemy (Tide) tem `hp = baseHp × anchorEnemyHpMultiplier` (default 3.0).
- Mis-focus penalty: `enemy_dmg × 1.5` em qualquer tick onde `focusIdx != tideIdx` E Tide vivo.
- `PreviewNextFloor` **deve** retornar a mesma Tide designation que a floor real terá (não um random).

### Asset shape (decisão tomada)
- Dungeon definitions vivem em CSV (`Balance/balance_dungeons.csv`), 1 row por dungeon. Loader popula um registry SO em runtime/edit-time. Designer edita CSV.
- Colunas: `id, displayName, sortIndex, themeColor, themeIconRef, focusRule, cardFocusRuleText, cardPressureHintText, unlockRule, enemyHpBase, enemyHpGrowth, enemyAtkBase, enemyAtkGrowth, enemyCountBase, enemyCountStep, goldBase, goldGrowth, floorClearTimeBase, floorClearTimePerFloor, anchorEnemyHpMultiplier, misFocusDamageMultiplier`.

### Não-fazer (out of scope v1)
- Spire (single-hero dungeon)
- Branching paths
- Hero synergy tags
- Modifiers (Volatile/Reflective/Necrotic/etc)
- Boss floors
- Dungeon difficulty tiers
- Per-dungeon currency
- Echoes/Sanctum/prestige

### `RunController.cs` (resumo das partes que precisam mudar)

```csharp
public class RunController : MonoBehaviour
{
    public TuningConfig config;  // SO global hoje — vai precisar ser per-dungeon

    public RunState State { get; private set; } = RunState.Idle;
    public int CurrentFloor { get; private set; } = 0;
    public int Gold { get; private set; } = 0;
    public int FocusEnemyIndex { get; private set; } = -1;
    public void SetFocus(int idx) { FocusEnemyIndex = idx; }
    public void ClearFocus() { FocusEnemyIndex = -1; }

    public List<HeroStats> Party { get; private set; } = new List<HeroStats>();

    public void StartRun(List<HeroData> partyData)  // ← vai virar StartRun(party, dungeon)
    {
        // ... PartyData = partyData; CurrentFloor = 0; Gold = 0; ...
    }

    public List<EnemyStats> BeginFloor()
    {
        CurrentFloor++;
        return BuildEnemiesForFloor(CurrentFloor);
        // ← também precisa designar Tide enemy se dungeon for Crypt
    }

    public List<EnemyStats> BuildEnemiesForFloor(int floor)
    {
        int count = EnemyCountBase + (Mathf.Max(0, floor - 1) / Mathf.Max(1, EnemyCountStep));
        int hp = Mathf.RoundToInt(EnemyHpBase * Mathf.Pow(EnemyHpGrowth, floor - 1));
        int atk = Mathf.RoundToInt(EnemyAtkBase * Mathf.Pow(EnemyAtkGrowth, floor - 1));
        var list = new List<EnemyStats>(count);
        for (int i = 0; i < count; i++)
            list.Add(new EnemyStats { hp = hp, atk = atk, def = 0 });
        return list;
        // ← Hollow precisa count maior + hp/atk customizado
        // ← Crypt precisa anchor com HP×multiplier no slot tideIdx
    }

    public FloorPreview PreviewNextFloor()
    {
        int floor = CurrentFloor + 1;
        var enemies = BuildEnemiesForFloor(floor);
        int seed = floor * 1000 + Party.Count;
        var sim = CombatResolver.Resolve(Party, enemies, FocusEnemyIndex, seed);
        // ← se Crypt, sim precisa receber tideIdx + misFocusMultiplier
        // ← determinismo: preview e run real precisam dar resultado idêntico
    }
}
```

## 4. PLANO DE IMPLEMENTAÇÃO PROPOSTO

1. **CSV → SO loader** minimalista em `Assets/Prototype/Scripts/Balance/CsvLoader.cs`. Lê `Balance/balance_general.csv` e `Balance/balance_dungeons.csv` em edit-time (via `[InitializeOnLoad]`) e popula SOs em `Assets/Prototype/Tuning/`. Roda em runtime fallback se SOs faltarem.
2. **`DungeonDefinition.cs`** SO: 1 SO por dungeon (gerado pelo loader). Campos = colunas do CSV.
3. **`DungeonRegistry.cs`** SO: lista de DungeonDefinitions, ordenada por sortIndex.
4. **`RunController.StartRun(party, dungeon)`**: parameteriza curve readers pelo dungeon ativo. Mantém defaults globais como fallback.
5. **`CombatResolver.Resolve(...)`** ganha parâmetro `DungeonRule` (struct: focusRule enum + tideIdx + misFocusMultiplier). Standard = today's behavior. NoFocus força focusIdx=-1. ForcedFocus aplica ×1.5 quando focus≠tide e Tide vivo.
6. **`MetaState`**: novos campos persistidos como List<string> + List<{id,int}> (JsonUtility-friendly), métodos `IsUnlocked(id)`, `GetDeepest(id)`, `RecordRunEnd(id, floor, gold)`, `MarkSeenJustUnlocked(id)`, `EvaluateUnlocks()`. Save bump para v2 com migração.
7. **Dungeon Select screen** novo no UIController (`Screen.DungeonSelect`). 3 cards horizontais. Lock card = shake. Tap unlocked = direct nav pro Roster com banner do dungeon.
8. **Combat UI**: focus button respeita `dungeon.focusRule`. Tide enemy ring (azul) em Crypt floors. Tooltip "Mis-focus penalty active" sutil quando aplicável.
9. **Bank/wipe** → `ReturnToDungeonSelect()` (substitui `ReturnToMainMenu`).
