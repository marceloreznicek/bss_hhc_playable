══════════════════════════════════════════════════════════════════
PROJECT CONTEXT
══════════════════════════════════════════════════════════════════

- **Engine:** Unity 6, C#. UI is uGUI (Canvas + RectTransform), procedurally built in `UIController.cs`.
- **Architecture pillar:** combat is **deterministic** and stat-driven. Combat resolution lives in `CombatResolver`/`RunController` and is decoupled from the UI. The UI only schedules animations on top of the resolver output. **This feature must NOT alter combat resolution or determinism.** It is purely a presentation/flow change between floors.
- **Floor flow today (no walk):**
  1. Player clicks DESCEND → `OnDescendClicked()` → `run.BeginFloor()` returns enemies → `BuildEnemyVisuals(enemies)` → `StartCoroutine(RunLiveCombat(enemies))`.
  2. `RunLiveCombat` ticks heroes/enemies in a strict turn-cycle until cleared/wiped → calls `run.ApplyFloorOutcome(...)`.
  3. On clear: UI shows DESCEND / BANK; an `AUTO DESCEND IN N` timer (`StartDescendTimer`) auto-clicks DESCEND if the player does nothing. On wipe: button becomes `RETURN`. On bank: `RunBanked` event fires.
- **No spatial scene yet.** Heroes and enemies are uGUI elements (`Image`/`RectTransform`) in `combatPanel`, anchored to rows (`heroRow` and `enemyRow`). Each visual has a `homeAnchoredPos` we can tween to/from. There is **no actual "room geometry"** — we re-use the same panel between floors.
- **Existing assets we can reuse:** `flashOverlay` Image already exists (used for damage flashes); we can repurpose for fade. Hero `Animator` has clips like `Idle`, `attack`, `Defeat`. **No `walk` animation clip is guaranteed in v1** — the proposal falls back to idle-slide (translate the idle sprite) until a walk clip is wired.
- **Prototype phase:** ~5 prototype scripts. No backwards-compat constraints. We can change struct shapes / add events freely.

v1 SCOPE STATEMENT (LOCKED — this is the contract)
══════════════════════════════════════════════════════════════════

1. **Trigger.** Sequence plays on every successful floor clear when the player chooses **Continue (Descend)** to the next floor. Wipe → no sequence (jump to RunOver UI). Bank → no sequence (jump to RunBanked UI). The Bank/Continue choice happens **before** the walk-out (so the moment of decision is preserved).

2. **Sequence (blocking — UI inputs disabled until sequence ends).**
   - **Walk-out (~0.5s):** all hero visuals translate upward off the top of `combatPanel` from their `homeAnchoredPos`. Speed = constant. Use a coroutine that lerps `anchoredPosition.y` outward.
   - **Fade-out (~0.25s):** `flashOverlay` fades from alpha 0 → 1 (black). Concurrent with the **last** part of walk-out OR sequenced after — implementation detail, but total walk-out + fade should land at ~0.75s.
   - **Room swap (instant, while screen is black):** existing enemy visuals are destroyed (`BuildEnemyVisuals` already does this on entry to next floor); next floor's enemies are built and placed at their `homeAnchoredPos`; hero visuals are repositioned to a "below-screen" entry point (their `homeAnchoredPos.y - offset`) so they can walk back in.
   - **Fade-in (~0.25s):** `flashOverlay` fades 1 → 0.
   - **Walk-in (~0.5s):** hero visuals translate from below-screen entry point up to their `homeAnchoredPos`. Constant speed.
   - **FIGHT banner (~0.6s hold, with brief punch-in scale at start):** large "FIGHT" Text overlay in the center of `combatPanel`. Pulse-in scale (e.g., 1.4 → 1.0 over 0.1s), hold ~0.4s, fade out ~0.1s.
   - **Combat begins:** after banner finishes, call existing `RunLiveCombat(enemies)` path. **No change** to combat ticking behavior.

3. **Total budget: ~2.1s** (walk-out 0.5 + fade 0.5 + walk-in 0.5 + banner 0.6). Numbers are tunable but should live as `const float` at top of UIController (NOT hardcoded inline).

4. **State guards (CRITICAL).**
   - `isPlayingFloor` must be `true` for the entire transition + combat (so DESCEND/BANK can't be clicked, and the auto-descend timer is stopped before sequence starts).
   - The auto-descend timer (`StopDescendTimer`) must be stopped at the moment the player commits to Descend — same as today.
   - `descendButton.interactable = false` and `bankButton.interactable = false` for the duration of the sequence — same as today during combat.
   - **No double-trigger:** if `isPlayingFloor == true`, `OnDescendClicked` early-returns (existing guard preserved).

5. **Dead party members.** Heroes with `hp <= 0` do NOT walk. They:
   - Stay slumped (current `Defeat` animation) and do NOT translate during walk-out — they fade out with the screen instead.
   - During room swap, they're repositioned at their `homeAnchoredPos` (off-screen-below entry is for alive heroes only).
   - They do NOT walk in. They appear at `homeAnchoredPos` already slumped when fade-in completes.
   - Their `hp = 0` state is preserved across floors (current behavior — `RunController.ApplyFloorOutcome` does not revive).

6. **Camera.** No camera movement. The "room change" is sold by the fade-to-black + the walk-out/walk-in deltas. We do NOT pan or stitch room geometry. (Out of scope; flagged for later.)

7. **Enemy entry.** Enemies for the next floor are **already in place** when fade-in completes. They do NOT walk in. The party walks toward static enemies, then "FIGHT" lands.

8. **Wipe path (UNCHANGED).** On wipe, no walk-out plays; existing `Flash(red)` + `WIPED on floor N` UI shows. Heroes do not walk anywhere on death.

9. **Bank path (UNCHANGED).** On Bank click, `run.Bank()` fires immediately, `OnRunBanked` event runs the existing flash + return-to-meta flow. No walk-out.

10. **Final-floor / dungeon-clear.** Out of scope for v1. The walk-out plays on every successful floor clear that leads to another floor. There is no "final floor" concept in current prototype; if a dungeon has finite floors in the future, the sequence will need a special-case skip — flag this as a known limitation, not a v1 task.

11. **Walk animation.** v1 falls back to **idle-slide** (translate the existing idle sprite). If the hero `Animator` has a `walk` state, switch to it during walk-out/walk-in and revert to `Idle` after; otherwise leave on `Idle` and just translate. No hard requirement on a walk clip in v1.

12. **Determinism.** This feature MUST NOT call into `RunController` for anything other than the existing `BeginFloor()` call (which it already does). The walk sequence is pure UI; it does not touch `Party`, `enemies`, RNG, or floor state. **Combat tick output is unchanged byte-for-byte.**

13. **Out of v1 scope:**
   - Camera pan / parallax / room geometry
   - Per-dungeon room art variety
   - Boss-floor unique entry
   - Walk animation clip authoring (idle-slide for v1)
   - Enemy walk-in / spawn animations
   - SFX (wire one event hook `OnFightBannerShown` for future audio; do not add audio code in v1)

```csharp
// Top of UIController class — new constants (tunable; live next to other timing consts)
const float WalkOutDuration   = 0.5f;
const float FadeOutDuration   = 0.25f;
const float FadeInDuration    = 0.25f;
const float WalkInDuration    = 0.5f;
const float FightBannerHold   = 0.4f;
const float FightBannerIn     = 0.1f;
const float FightBannerOut    = 0.1f;
const float WalkOffsetY       = 600f;   // pixels off-screen for walk start/end

// New event for future audio hookup (v1 has no subscribers)
public event System.Action OnFightBannerShown;

// New: FIGHT banner Text built once in BuildCombat() alongside flashOverlay.
Text fightBanner; // built once, disabled by default

// MODIFIED: OnDescendClicked replaces the existing body
void OnDescendClicked()
{
    if (isPlayingFloor) return;
    StopDescendTimer();
    ClearDamageLayer();

    // Disable buttons immediately so the player can't double-click during transition
    descendButton.interactable = false;
    bankButton.interactable = false;

    // CurrentFloor > 0 means we just cleared a floor — play the walk transition.
    // CurrentFloor == 0 means this is the FIRST descend of the run — skip walk-out
    // (heroes haven't been "in" any room yet). Floor 1 still gets fade-in + FIGHT banner.
    bool playWalkOut = run.CurrentFloor > 0;
    StartCoroutine(FloorTransitionThenCombat(playWalkOut));
}

IEnumerator FloorTransitionThenCombat(bool playWalkOut)
{
    isPlayingFloor = true;          // lock for the entire transition + combat
    try
    {
        // 1. Walk-out (only if we cleared a floor)
        if (playWalkOut)
            yield return WalkOutAlive();

        // 2. Fade to black
        yield return FadeOverlay(0f, 1f, FadeOutDuration);

        // 3. Room swap (while black)
        var enemies = run.BeginFloor();
        BuildEnemyVisuals(enemies);
        UpdateFocusVisuals();
        UpdateCounters();
        // Reposition alive heroes below-screen for walk-in
        for (int i = 0; i < heroVisuals.Count && i < run.Party.Count; i++)
        {
            var hv = heroVisuals[i];
            if (hv.root == null) continue;
            if (run.Party[i].hp <= 0)
            {
                // Dead hero — stay at home, slumped sprite already set last floor
                hv.root.anchoredPosition = hv.homeAnchoredPos;
            }
            else
            {
                hv.root.anchoredPosition = hv.homeAnchoredPos + new Vector2(0, -WalkOffsetY);
                if (hv.animator != null) hv.animator.Play("Idle", 0, 0f);
            }
        }

        // 4. Fade in
        yield return FadeOverlay(1f, 0f, FadeInDuration);

        // 5. Walk in (alive heroes only)
        yield return WalkInAlive();

        // 6. FIGHT banner
        yield return ShowFightBanner();

        // 7. Hand off to combat (UNCHANGED resolver path)
        // (isPlayingFloor stays true — RunLiveCombat asserts it on entry)
        yield return RunLiveCombat(enemies);
    }
    finally
    {
        // RunLiveCombat already toggles isPlayingFloor = false on exit;
        // belt-and-suspenders in case of exception during the transition itself.
        isPlayingFloor = false;
    }
}

IEnumerator WalkOutAlive()
{
    float t = 0f;
    // Snapshot start positions for alive heroes
    var starts = new Vector2[heroVisuals.Count];
    for (int i = 0; i < heroVisuals.Count; i++)
        starts[i] = heroVisuals[i].root != null ? heroVisuals[i].root.anchoredPosition : Vector2.zero;

    while (t < WalkOutDuration)
    {
        t += Time.deltaTime;
        float u = Mathf.Clamp01(t / WalkOutDuration);
        for (int i = 0; i < heroVisuals.Count && i < run.Party.Count; i++)
        {
            if (run.Party[i].hp <= 0) continue; // dead heroes don't walk
            var hv = heroVisuals[i];
            if (hv.root == null) continue;
            hv.root.anchoredPosition = starts[i] + new Vector2(0, WalkOffsetY * u);
        }
        yield return null;
    }
}

IEnumerator WalkInAlive()
{
    float t = 0f;
    var starts = new Vector2[heroVisuals.Count];
    for (int i = 0; i < heroVisuals.Count; i++)
        starts[i] = heroVisuals[i].root != null ? heroVisuals[i].root.anchoredPosition : Vector2.zero;

    while (t < WalkInDuration)
    {
        t += Time.deltaTime;
        float u = Mathf.Clamp01(t / WalkInDuration);
        for (int i = 0; i < heroVisuals.Count && i < run.Party.Count; i++)
        {
            if (run.Party[i].hp <= 0) continue;
            var hv = heroVisuals[i];
            if (hv.root == null) continue;
            hv.root.anchoredPosition = Vector2.Lerp(starts[i], hv.homeAnchoredPos, u);
        }
        yield return null;
    }
    // Snap to exact home (avoid float drift)
    for (int i = 0; i < heroVisuals.Count && i < run.Party.Count; i++)
    {
        if (run.Party[i].hp <= 0) continue;
        if (heroVisuals[i].root != null)
            heroVisuals[i].root.anchoredPosition = heroVisuals[i].homeAnchoredPos;
    }
}

IEnumerator FadeOverlay(float fromAlpha, float toAlpha, float duration)
{
    if (flashOverlay == null) { yield break; }
    // Force black for the room-swap fade (overlay is reused for damage flashes; restore color afterward is fine
    // because every damage flash sets its own color)
    var c = flashOverlay.color; c.r = 0f; c.g = 0f; c.b = 0f; c.a = fromAlpha;
    flashOverlay.color = c;
    float t = 0f;
    while (t < duration)
    {
        t += Time.deltaTime;
        float u = Mathf.Clamp01(t / duration);
        c.a = Mathf.Lerp(fromAlpha, toAlpha, u);
        flashOverlay.color = c;
        yield return null;
    }
    c.a = toAlpha; flashOverlay.color = c;
}

IEnumerator ShowFightBanner()
{
    if (fightBanner == null) yield break;
    fightBanner.gameObject.SetActive(true);
    var rt = fightBanner.rectTransform;

    // Punch-in
    float t = 0f;
    while (t < FightBannerIn)
    {
        t += Time.deltaTime;
        float u = Mathf.Clamp01(t / FightBannerIn);
        rt.localScale = Vector3.Lerp(new Vector3(1.4f, 1.4f, 1f), Vector3.one, u);
        var col = fightBanner.color; col.a = u; fightBanner.color = col;
        yield return null;
    }
    rt.localScale = Vector3.one;
    var c = fightBanner.color; c.a = 1f; fightBanner.color = c;

    OnFightBannerShown?.Invoke();

    // Hold
    yield return new WaitForSeconds(FightBannerHold);

    // Fade out
    t = 0f;
    while (t < FightBannerOut)
    {
        t += Time.deltaTime;
        float u = Mathf.Clamp01(t / FightBannerOut);
        var col = fightBanner.color; col.a = 1f - u; fightBanner.color = col;
        yield return null;
    }
    fightBanner.gameObject.SetActive(false);
}
```
